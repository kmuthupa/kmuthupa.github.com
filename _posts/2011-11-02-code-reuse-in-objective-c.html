--- layout: post title: Code Reuse in Objective C tags: - General status: publish type: post published: true meta: _edit_last: '13680702' jabber_published: '1320226749' _wpas_done_twitter: '1' --- One of the trivial ways to enable code reuse in traditional programming platforms like C++, C# or VB is building dynamic libraries that are compiled binaries that can be shared with different consumers who can embed these libraries into their application's code and derive use out of them. These dynamic libraries (or DLLs) are linked with their host's code at run-time. The ability to use/link dynamic libraries is missing in Apple's Objective C platform for whatever reason. The alternative that the platform gives us (that I put to use recently) is building static libraries. Interestingly, this is probably the only way to enable code sharing/reuse in Objective C. This thing was pretty new to me having been used to the dynamic way for a long time. Â Xcode makes building static libraries a piece of cake. It is even possible to develop a library that would contain UI components (like Cocoa-touch views/controllers etc) and have an application make use of these components via the shared static library. This article goes over the process of developing a simple static library in Objective C using Xcode.
